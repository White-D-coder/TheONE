# Engineer OS: Speaking Practice Engine

## Purpose
To turn communication from a vague attribute into a measurable, trainable skill.

## Session Workflow
1. **Prompt Generation**: AI selects a question (e.g., "Explain how a B-tree works to a 5-year-old").
2. **Recording**: User speaks into the app.
3. **Transcription**: Speech-to-Text converts audio to a transcript.
4. **AI Scoring**: Analyzes the transcript across 5 pillars:
   - **Clarity**: Is the message easy to understand?
   - **Conciseness**: Are there unnecessary filler words (um, ah, like)?
   - **Structure**: Is there a clear Intro -> Body -> Conclusion?
   - **Technical Accuracy**: Is the content correct?
   - **Confidence**: Analyzes pacing and tone (via transcript hints).

## Drill Types
- **The Elevator Pitch**: 30 seconds to sell an idea.
- **The Deep Dive**: 5 minutes to explain a project.
- **The Mock Interview**: Behavioral question response.
- **The Demo Narration**: Explaining a feature while showing it.

## Analytics
- **Filler Word Tracker**: Graphs the frequency of "um" over time.
- **Clarity Trend**: Shows if the user is becoming more coherent.
- **Difficulty Progression**: Unlocks harder questions as the user improves.
