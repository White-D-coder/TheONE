# Engineer OS: Project Execution & Evidence Vault

## Purpose
To move from "thinking about projects" to "shipping evidence". The Evidence Vault is the ground truth for all career growth.

## Project Stages
1. **Idea**: Raw concept with target users/problems.
2. **Building**: Active development (commits, screenshots).
3. **Shipped**: Deployed link, README, and public post.

## Evidence Types
Evidence is categorized by **Strength**:
- **WEAK**: "I finished a tutorial" (Self-reported).
- **MEDIUM**: "I fixed a bug" (Commit link).
- **STRONG**: "I shipped a feature" (Merged PR, Live URL).
- **ELITE**: "Others are using it" (GitHub Stars, Client Testimonial, Users).

## The Vault
The Vault is a searchable repository of every technical artifact created by the user:
- Github Repos/Commits.
- Deployed URLs (Vercel, Railway).
- Video Demos (Loom links).
- Technical Articles (Medium, Dev.to).
- Certificates and Contest Ranks.

## Vault Actions
- **Retrieve for Resume**: AI finds the 3 strongest evidences for a specific job application.
- **Convert to Content**: "You just fixed a complex SQL bug. Want to draft a LinkedIn post about it?"
- **Project Pulse**: AI detects if a project has stalled and suggests a "Minimal Shippable Increment".
